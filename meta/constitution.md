# PCOS 宪法（V1）

这份宪法用于约束 PCOS 的长期演进方式：确保系统可维护、可审计、可扩展，并避免陷入过度工程。

## 基本分层

- **capture（原始层）**：记录当下语境与未经验证的想法，不追求结构化。
- **assets（资产层）**：沉淀可复用、可引用、可迭代的知识单元（Markdown）。
- **machine（机器层）**：用 JSON/Schema/Index 表达可计算结构，为脚本与 Agent 提供稳定接口。

## 核心原则

1. **人机双轨**
   - 人类层优先可读、可维护；机器层优先稳定、可验证。
2. **简洁优先**
   - V1 只保留必要结构；任何新增格式/目录都要能说明“解决了什么问题”。
3. **插件化扩展**
   - 高级能力以独立脚本/模块加入；避免把某个工具或服务写成系统前提。
4. **Git 即真理**
   - 所有关键产出必须可 diff、可回滚；不要把关键知识只放在不可追踪的外部系统。
5. **自动化友好**
   - 先定义输入/输出与 schema；自动化产物尽量可重建，避免手工与生成物混杂。

## 资产质量门槛（写入 `assets/` 前自检）

- 能一句话说明“这条资产解决什么问题”
- 写清边界（适用/不适用、前提、风险）
- 重要结论可追溯（来源链接或 capture 路径）
- 不追求完美：允许先写 V1，再通过 Git 演进
