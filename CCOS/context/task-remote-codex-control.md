## 需求概述

实现一个跨平台远程控制能力：当用户离开电脑时，可通过手机在异地网络（包含移动网络）监督 PC 上的 Codex AI 办公任务、发送提示词并查看执行状态。首期优先 Web 控制台，后续迁移/扩展到小程序。

## 验收标准（MVP）

1. 手机端登录后可查看在线 PC 设备及任务状态。
2. 可向指定任务发送提示词并在 10 秒内到达 PC Agent。
3. 可实时查看任务日志流（含历史消息回看）。
4. 可执行任务控制（停止/重试/继续）。
5. 跨网络可用，不依赖同一局域网。
6. 全链路鉴权、操作审计、命令白名单有效。

## 成熟基线方案

1. 架构：`手机 Web/小程序 -> 控制后端 -> PC Agent -> Codex Runtime`。
2. 通信：手机到后端使用 HTTPS/WSS；PC Agent 主动外连 WSS。
3. 控制策略：优先“任务级远程控制”，不在首期做远程桌面。
4. 数据层：PostgreSQL（任务/审计）+ Redis（会话/消息）。

## 更优备选方案（后续）

1. 增加 WebRTC 数据通道和可选屏幕流，用于高级监管与协作。
2. 增加团队协作权限模型、审批流与风险命令二次确认。

## 依赖关系

1. 控制协议先于 Agent/后端实现。
2. Agent 与后端可并行，均依赖协议冻结版本。
3. Codex Adapter 完成后再联调 Web 控制台。
4. Web MVP 稳定后再推进小程序，避免双端并行放大风险。

## 风险与缓解

| 风险 | 影响 | 缓解方案 |
|---|---|---|
| Codex 接口不稳定 | 无法可靠下发提示词 | 抽象 `TaskAdapter`，先支持 CLI/PTY |
| 远程执行高风险 | 安全事故 | 强鉴权、最小权限、白名单、审计 |
| 移动网络抖动 | 指令丢失/重复 | ACK、重试、幂等 `command_id` |
| 多端并发操作 | 会话冲突 | 会话锁、租约、序列号 |

## 用户已确认决策

1. 不采用远程桌面方案，保持“任务级控制”路线。
2. 方案优先贴近 OpenClaw 的网关思路，本地实现 Codex 适配接口接入网关。
3. 本机无公网 IP 场景优先使用“本机主动外连”的组网方式（优先 Tailscale）。
4. 资源策略倾向“云桥接控制本地 PC”，将计算压力保留在本机，降低云资源消耗。
